# User Stories


## 1 View paginated list of posts

As a site user, I can view a paginated list of posts so that I can select which post I want to view.


### Acceptance criteria

:ballot_box_with_check: Given more than one post in the database, these multiple posts are listed.  
:ballot_box_with_check: When a user opens the main page a list of posts is seen.  
:ballot_box_with_check: Then the user sees all post titles with pagination to choose what to read.


## 2 Open a post

As a Site User, I can click on a post so that I can read the full text.


### Acceptance criteria

:ballot_box_with_check: When a blog post title is clicked on a detailed view of the post is seen.


## 3 View comments

As a Site User / Admin I can view comments on an individual post so that I can read the conversation.


### Acceptance criteria

:ballot_box_with_check: Given one or more user comments the admin can view them.  
:ballot_box_with_check: Then a site user can click on the comment thread to read the conversation.


## 4 Account registration

As a Site User I can register an account so that I can comment on a post.


### Acceptance criteria

:ballot_box_with_check: Given an email a user can register an account.  
:ballot_box_with_check: Then the user can log in.  
:ballot_box_with_check: When the user is logged in they can comment.


## 5 Comment on a post

As a Site User I can leave comments on a post so that I can be involved in the conversation


### Acceptance criteria

:ballot_box_with_check: When a user comment is approved  
:ballot_box_with_check: Then a user can reply  
:ballot_box_with_check: Given more than one comment then there is a conversation thread


## 6 Modify or delete comment on a post

As a Site User I can modify or delete my comment on a post so that I can be involved in the conversation


### Acceptance criteria

:ballot_box_with_check: Given a logged in user, they can modify their comment  
:white_square_button: Given a logged in user, they can delete their comment


## 7 Manage posts

As a Site Admin I can create, read, update and delete posts so that I can manage my blog content


### Acceptance criteria

:ballot_box_with_check: Given a logged in user, they can create a blog post  
:ballot_box_with_check: Given a logged in user, they can read a blog post  
:ballot_box_with_check: Given a logged in user, they can update a blog post  
:ballot_box_with_check: Given a logged in user, they can delete a blog post


## 8 Create drafts

As a Site Admin I can create draft posts so that I can finish writing the content later


### Acceptance criteria

:ballot_box_with_check: Given a logged in user, they can save a draft blog post  
:ballot_box_with_check: Then they can finish the content at a later time


## 9 Approve comments

As a Site Admin I can approve or disapprove comments so that I can filter out objectionable comments


### Acceptance criteria

:ballot_box_with_check: Given a logged in user, they can approve a comment  
:ballot_box_with_check: Given a logged in user, they can disapprove a comment


## 10 Read about the site

As a site user I can click on the "About" link so that read about the site


### Acceptance criteria

:ballot_box_with_check: When the "About" link is clicked, the about text is visible


## 11 Add and update the "About" text

As a site admin, I can create or update the "About" page content so that it is available on the site.


### Acceptance criteria

:ballot_box_with_check: The "About" app is visible in the admin panel.


## 12

As a Potential Collaborator I can fill in a contact form so that I can submit a request for collaboration.


## 13

As a Site Owner I can store collaboration requests in the database so that I can review them.


## 14

As a Site Owner I can mark collaboration requests as "read" so that I can see how many I still need to process.
